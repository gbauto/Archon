4. The Sequence — 9 HeyReach Steps

Every step maps to a NEPQ phase (Miner). The prospect reaches the meeting ask through their own problem-awareness, not a pitch.

Step 1 — View Profile (Day 0)

  • Action: View Profile (automated, zero-message)
  • Purpose: Surface Jason as a named profile viewer in the prospect's LinkedIn notifications

Step 2 — Like Recent Post (Day 1, conditional)

  • Action: Like latest post (automated, only if the prospect posted within the last 30 days; skip if post is political or vendor-war)
  • Purpose: Second neutral warming touch. Shows up in their activity feed.

Step 3 — Connection Request (Day 2) — NEPQ: Connection

Action: Send Connection Request with Note. Character limit: 300.

Copy (279 chars):

Dr. [LastName] — I run an advisory that rebuilds RCM + workflow for 1–8 provider independent practices. Not pitching. Following independent physician owners who are the audience for our content. Happy to connect.

Move: peer-frame ("I run an advisory"), permission-based ("not pitching"), reverse-ask ("audience for our content" — you're offering future value, not asking for time).

Step 4 — Post-Accept DM #1 · Situation Question (Day 0 after accept) — NEPQ: Situation

Action: Send Message. Trigger: immediately on accept.

Copy:

Appreciate the connect, Dr. [LastName].

Quick one if it's useful: across the last [FILL: N] independent practices we audited (1–8 providers), the honest answer to "how are denied claims being caught before they age past 90 days?" was usually some version of "my biller watches for them, I think." Median leakage sitting in that gap: $[FILL: exact audit figure — e.g. 147,200]/yr.

How is that handoff handled at [Practice Name] today?

Not a pitch — just curious where you land on the pattern.

— Jason

Move: Voss calibrated question · Vaynerchuk give-first · Dunford (alternative implicit: not a software problem) · Miller (physician is the hero of the question).

Fill rule (hard): do not ship with [FILL: …] in place. Pull real audit data or rotate to a pattern-only variant (no figure). Never fabricate.

Step 5 — Post-Accept DM #2 · Problem-Awareness Question (Day 4 after accept) — NEPQ: Problem Awareness

Action: Send Message. Delay: 4 days from Step 4. Condition: no reply.

Copy:

Dr. [LastName] —

Following up on the denial-queue note, one more curious question:

When a claim does slip past the 90-day mark at [Practice Name], how does that usually show up on your end — do you see it on a report, does the biller raise it, or does it just quietly land in a write-off bucket at quarter-close?

Asking because in [FILL: N] of the last [FILL: N] audits we ran it landed in the third bucket — which means nobody on the team actually owns the number. Not a software problem. Not a biller problem. A handoff problem.

If that's even partly live at [Practice Name], worth a look. If not, I'll be quiet.

— Jason

Move: Multi-option calibrated question where the third option is the diagnostic trap · three Dunford alternatives named · Miner permission-based exit.

Step 6 — Post-Accept DM #3 · Consequence + First Meeting Ask (Day 9 after accept) — NEPQ: Consequence + Commitment

Action: Send Message. Delay: 5 days from Step 5. Condition: no reply.

Copy:

Dr. [LastName] —

Last operational question before I step back, because I don't want to over-poke:

If that handoff gap stays the way it is for another 12 months at [Practice Name], what does that look like — is it "same cash flow, roughly", or is it the kind of drift that quietly eats the year?

I'm asking because the practices that ran our 30-day audit in January looked almost identical to the ones that waited until Q3 — except the Q3 ones were rebuilding against a bigger hole.

If it's worth 20 minutes to see the actual number for [Practice Name], I'll send a calendar link. If the queue is clean, ignore me and I'll drop off.

— Jason

Move: Voss consequence question with two framings (self-soothing vs. honest — prospect's own mind surfaces the consequence) · no-oriented close ("ignore me and I'll drop off") · Miner permission-based.

Step 7 — Post-Accept DM #4 · Voice Note · Solution-Awareness (Day 14 after accept) — NEPQ: Solution Awareness · CREATIVE

Action: Send Voice Note (HeyReach native — 30–45 sec MP3, < 5MB). Delay: 5 days from Step 6. Condition: no reply.

Text caption / fallback DM (send together with the voice note):

Dr. [LastName] — voice note above, ~40 sec. One Solution-Awareness question, no ask. If voice notes aren't your thing, the question is:

If you could redesign the front-desk-to-billing handoff from scratch, what would need to be true for you to trust the month-end number?

No reply needed — just wanted to leave it with you.

— Jason

Move: Miner solution-awareness (prospect articulates their own fix) · pattern interrupt (voice vs. text in LinkedIn inbox) · explicit no-ask framing (drives reply paradoxically).

5. The "Captivating Hook" (Brand Interpretation)

Per PMC Brand Guide v2 — Quiet Luxury / Hermès of Healthcare Advisory — fear hooks and attention-grabs are explicitly retired. The PMC equivalent of a hook is the calibrated question exposing a blind spot, delivered in clinical-peer tone. Every step's opening sentence is engineered for this function.

What a PMC hook looks like (from this sequence): - "How are denied claims being caught before they age past 90 days?" (Step 4 — situation) - "How does that usually show up on your end — do you see it on a report, does the biller raise it, or does it just quietly land in a write-off bucket?" (Step 5 — the third option is the trap) - "Is it the kind of drift that quietly eats the year?" (Step 6 — consequence framing) - "If you could redesign that handoff from scratch, what would need to be true?" (Step 7 — solution-awareness)

What a PMC hook never is: - "Your front desk is leaking $200K/year" (retired — hero-posture, rounded number, fear hook) - "You're 90 days away from a completely different practice" (retired — AI-generated, generic, three-paragraph template) - Anything with exclamation marks, emoji, "Drop a comment," or "Hey [FirstName]!"

6. ICP Variant Swap Matrix

Default sequence above targets the generic 1–8 provider independent practice. For specialty-concentrated lists, swap the calibrated questions, labeling statements, Dunford alternatives, proof anchors, and subject line bank per variant below. Sequence architecture (9 steps, cadence, NEPQ phase per step) stays identical.

Variant A — Functional / Integrative Medicine

  • Calibrated opener (Step 4): How are you currently measuring the margin on your membership — separately from cash visits and supplement revenue?
  • Labeling (Step 5): It sounds like the practice grew around the medicine you wanted to practice, but the pricing and membership structure was never redesigned as the patient base changed.
  • Dunford alternatives to name: Not a functional-med marketing agency · not another provider hire · not a membership-software vendor
  • Subject bank: Superbill leakage pattern · Membership pricing audit · Your supplement margin · Cash vs. insurance gap
  • Proof anchor: [FILL: pull from recent functional-med audit — median cash-pay collections gap, median supplement margin delta, or membership repricing uplift]

Variant B — Chiropractic

  • Calibrated opener (Step 4): How is your care-plan completion rate tracked — and who sees it when a patient drops off between visit 6 and visit 12?
  • Labeling (Step 5): It sounds like the front desk is selling the plans fine, but nobody owns the retention conversation between visit 6 and visit 12 — which is where most of the package revenue actually disappears.
  • Dunford alternatives to name: Not a chiro CRM (Chirotouch / Genesis / Jane) · not another associate hire · not a chiro coach on a monthly group call
  • Subject bank: Care plan drop-off · Visit 6 to 12 gap · Medicare doc pattern · Chiro denial codes
  • Proof anchor: [FILL: pull from recent chiro audit — median package completion rate, denial-code pattern, or retention uplift after SOP rebuild]

Variant C — Plastics / Medspa / Aesthetics

  • Calibrated opener (Step 4): How is your consult-to-procedure conversion tracked — and what happens to the consults that don't book same-day?
  • Labeling (Step 5): It sounds like marketing is doing its job — consults come in — but the conversion to booked procedure is where the real leakage is, and nobody owns the 30-day followup on the ones who leave "thinking about it."
  • Dunford alternatives to name: Not an aesthetics marketing agency · not a medspa software vendor (Aesthetic Record / PatientNow) · not another injector hire
  • Subject bank: Consult-to-booking gap · Your package unit cost · 30-day consult followup · No-show policy math
  • Proof anchor: [FILL: pull from recent aesthetics audit — median consult-to-booking conversion, package margin after discount stack, or no-show rate delta after deposit policy]

Variant D — Longevity / Anti-Aging / Concierge

  • Calibrated opener (Step 4): How are you measuring margin on your lab panels separately from the rest of the membership?
  • Labeling (Step 5): It sounds like the practice is profitable in aggregate, but you've never separated which service lines subsidize which — so every pricing decision is made without knowing which part of the business is actually carrying the margin.
  • Dunford alternatives to name: Not a concierge consultancy · not a lab rep (Quest / Genova / Labcorp) · not a fractional CFO
  • Subject bank: Lab panel margin · Membership tier audit · Service line rollup · Executive physical LTV
  • Proof anchor: [FILL: pull from recent concierge audit — median lab panel margin spread, membership repricing delta, or service-line profitability breakdown]

Variant selection rule

Pick the variant that matches >60% of leads in the list. If the list is truly mixed (no dominant specialty), split into sub-lists by specialty before push — one variant per sub-list, never mix within a sub-list.

8. Webhooks (Automation Spine)

Register five events via POST /api/public/webhook/Create (exact path TBD — see §14 troubleshooting). Destination URL is either a Zapier catch hook, Make.com scenario, or n8n webhook — depending on which the operator has.

Event Handler action
ACCEPTED_INVITE Update master_leads.csv: sequence_status = in-heyreach, tag connected, last_touched_at = now
MESSAGE_REPLIED Pause sequence for that lead. sequence_status = replied. Route reply text into second-brain/intelligence/inbox/YYYY-MM-DD-replies.md. Email operator via Gmail MCP (batched hourly to avoid noise).
CONNECTION_REJECTED sequence_status = suppressed, suppression_reason = rejected-connect. Add to master suppression list (cross-channel).
DISCONNECT Hard suppress across PMC and BRT channels. Master suppression list update. Never contact again.
MEETING_BOOKED sequence_status = booked, tag meeting-booked. Immediate operator alert (bypasses batching).

Rate limit on operator alerts

No more than 1 batched email per hour except MEETING_BOOKED (immediate). Prevents inbox fatigue.

9. Reply Playbook (In-Inbox Objection Handling)

When a DM comes back, HeyReach pauses the sequence for that lead. The operator responds in-HeyReach (threaded, tracked) using the patterns below. Never argue. Never rebut. Always label.

Reply category Example incoming Response pattern
Positive — curious "tell me more" / "what do you do exactly?" One-paragraph answer that names the deliverable (RCM + workflow + team systems, 30-day audit). One calibrated follow-up question. No deck, no pitch.
Positive — book "sure, send a time" / "happy to chat" Calendar link. Stop sequence immediately. No additional context.
Objection — not now "not a priority right now" / "maybe Q1" Label: "It sounds like the timing's not live — which makes sense. If the denial queue or front-desk handoff becomes a priority, I'll be here." Pause 30 days. Resume at Step 8 (reverse-pressure) only.
Objection — we're fine "we don't need this" / "we're covered" Label: "Totally fair. Most practices I talk to have the clinical side locked down — the question is usually whether the operational math is as clean as it feels. If it ever isn't, you know where to find me." No rebuttal.
Objection — have someone "we already have a consultant" / "working with X" Label: "Good — means the problem is named. My work usually picks up where project consultants leave off, once the deck is done. If that phase ever comes, happy to compare notes." No competitive attack.
Generic inbox / auto-reply "This inbox is not monitored" Drop from sequence. Mark bounced-generic. Add to suppression.
Curious but vague "what would this look like?" Send the one-pager. Resume sequence at next scheduled step.
Angry / unsubscribe "stop messaging me" / vulgar Honor immediately. Cross-channel suppression. Apologize once, briefly. Close loop.

Discovery-call bridge

Any positive reply that progresses to a 20-minute call — run the discovery per discovery-banks (calibrated-question bank + labeling-statement bank). Target: one label per call hits "that's right." If under 50% hit rate, the label is wrong — iterate.

Track in HeyReach native reporting + Notion/Sheets rollup. Reply rate is a vanity metric — do not optimize for it.

DM→reply % replies / accepted-connections 12–20% Step 4 + 5 calibrated-question quality
Reply→meeting % meetings booked / replies received 25–35% Reply-playbook execution quality
Meetings per 100 connections sent end-to-end funnel 1.5–3.0 The only number that matters

When to iterate

  • Connection acceptance <25% for 2 weeks → connection note copy is off, or the list ICP is wrong. Re-target.
  • DM→reply <8% for 2 weeks → Step 4 calibrated question isn't pinging. Rewrite per ICP variant.
  • Meetings per 100 connections <1.0 at 30 days → whole funnel needs a teardown. Do not scale.

11. Suppression + Dual-Brand Guardrails (Legal-Grade)

The three hard rules

  1. Never send PMC and BRT messaging from the same mailbox to the same prospect. If in PMC pipeline, no BRT. If in BRT pipeline, no PMC. Cross-pollination happens through the advisory relationship (Jason in a call), never via an automated channel.
  2. The PMC → BRT bridge is always the advisory relationship, never the email. A PMC client with a clinical retention problem hears about BRT in an advisory conversation. The advisor's credibility transfers.
  3. Sender accounts are brand-specific. jid5274@gmail.com is the PMC persona on LinkedIn. The BRT LinkedIn account is separate. Never cross-post.

Suppression pipeline (runs before every push — scripts/master_leads_validate.py enforces)

  1. Schema compliance — required fields present
  2. LinkedIn URL validity — must match https://www.linkedin.com/in/... pattern for HeyReach
  3. Apollo Sent dedup — sequence_status ∈ {in-apollo, replied-apollo} → REJECT
  4. BRT active campaigns dedup — cross-file check against BRT lead files → REJECT + legal-grade flag
  5. Master suppression dedup — sequence_status = suppressed → REJECT
  6. HeyReach existing state dedup — already in another active list → REJECT
  7. LinkedIn 1st-degree check — if already connected, route past Step 3 (connection request) but include
  8. Rate-budget check — max 18 new connections/day → truncate batch if over

Any legal-grade flag (BRT overlap) blocks the live push — operator must resolve manually.

Master suppression list columns

linkedin_url, email, reason, suppressed_at, brand_of_original_contact, channel_of_original_contact

14. Troubleshooting + Kill Switches

Common issues

Symptom Likely cause Fix
401 Unauthorized from HeyReach REST API key rotated or typo Regenerate in HeyReach UI → Settings → API Keys; update env var
Campaign won't start isActive: false on LinkedIn account Toggle active in HeyReach UI
Leads added but not appearing in campaign List-to-campaign attach failed silently HeyReach UI → campaign → Leads tab; retry attach via AddLeadsFromListV2

Kill switches (in order of severity)

  1. Pause one lead's sequence. HeyReach → Leads → Stop in Campaign. Use when a reply comes in that needs human follow-up.
  2. Pause the whole campaign. HeyReach → Campaigns → Pause. All sends stop immediately. Use when: metrics go sideways, brand damage signal, unexplained error spike.
  3. Revoke the HeyReach API key. HeyReach → Settings → API Keys → Revoke. Use if credential leakage suspected.
  4. Disconnect LinkedIn account. HeyReach → Linked Accounts → Disconnect. Cuts all sending from this account. Use if LinkedIn flags the account for suspicious activity.
  5. Revert master_leads.csv. git checkout <sha> -- second-brain/contacts/master_leads.csv. Use if a bad push corrupted state.
